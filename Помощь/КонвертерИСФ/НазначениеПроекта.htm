Назначение проекта "Конвертация"

Общие сведения

Проект предназначен для переноса (преобразования, конвертации) данных, накопленных при работе с прикладной подсистемой "Зарплата-Универсал" (разработчик - ООО "ИСФ <Консультант>"). Указанная прикладная подсистема реализована на к платформе "Турбо Бухгалтер 6.9".

Перед началом конвертации убедитесь, что импортируемая база данных входит в состав текущей рабочей информационной базы как внешняя БД. Конвертацию невозможно выполнить из локальных версий программы "Турбо Бухгалтер 6.9", работающих с форматом СУБД Paradox. Для выполнения импорта в этом случае требуется предварительный перенос данных в любую сетевую версию "Турбо Бухгалтер 6.9", работающую с любой версией СУБД MS SQL или Oracle!

Интерфейс

Пользовательский интерфейс проекта представляет собой форму (бланк) с таблицей выполнения этапов конвертации. Этапы конвертации объединены в группы. Запуск выполнения каждой группы этапов осуществляется с помощью группы кнопок "Запуск выполнения".

Прядок выполнения конвертации (общие принципы)

Большинство строк в таблице конвертации имеют кнопку обзора ("..."). Кнопка предназначена для вызова диалога настройки и (или) оценки результатов выполнения этапа конвертации. Если в диалоге настройки присутствует кнопка "Применить", то она запускает процесс выполнения выбранного этапа.
До подачи команды на выполнение группы все этапы, входящие в группу, должны быть настроены. При неполном выполнении этапа допускается возврат к его настройке (донастройке) и повторном выполнении.

Запуск выполнения групп этапов конвертации должн выполняться последовательно, нарушение порядка не допускается. Команда на выполнение каждой следующей группы этапов конвертации доступна после полного завершения предыущей.

Каждый этап конвертации имеет свои особенности, отраженные в "Помощи". Полностью автоматизировать процесс переноса данных невозможно из-за различия в архитектуре прикладных решений.
